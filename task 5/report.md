### 1. Telescoping Constructor

Класс `HttpRequest` содержит множество параметров в одном конструкторе:

* трудно поддерживать
* сложно расширять
* низкая читаемость

---

### 2. Primitive Obsession

Middleware управляется через битовую маску:

```python
middleware_flags & 0x01
```

Проблемы:

* неочевидная логика
* слабая типизация
* сложно расширять

---

## Применённые паттерны

### Builder

Реализован `HttpRequestBuilder` с fluent-интерфейсом:

* пошаговая настройка
* валидация на этапе build()
* улучшенная читаемость

---

### Decorator

Middleware реализованы как декораторы:

* каждый middleware оборачивает следующий
* независимость компонентов
* возможность произвольной комбинации

---

### Null Object

Реализован `NullMiddleware`:

* отсутствие проверки на None
* безопасное поведение по умолчанию

---

## Архитектура после рефакторинга

* HttpRequest создаётся через Builder
* Middleware строятся как цепочка декораторов
* execute_request заменён на pipeline
