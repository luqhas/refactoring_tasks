# Задания

- Применить паттерн State: создать иерархию классов состояний с общим интерфейсом.
- Каждое состояние должно обрабатывать все возможные события: insert_coin, select_product, cancel, refill.
- Реализовать переходы между состояниями без использования условных операторов в контексте.
- Добавить новое состояние MaintenanceMode без изменения существующих классов (демонстрация принципа OCP).
- Разработать диаграмму состояний (State Diagram) в нотации UML.
- Написать интеграционные тесты, проверяющие корректность всех возможных переходов между состояниями.
